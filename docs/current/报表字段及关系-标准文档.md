## 2. 计算层：字段关系与公式

### 超级直播报表计算公式

- `ROI = 总成交金额 / 花费`  
- `直接ROI = 直接成交金额 / 花费`  
- `间接ROI = (总成交金额 - 直接成交金额) / 花费`
- `有效观看率 = 有效观看次数 / 观看次数 × 100%`  
- `有效观看成本 = 花费 / 有效观看次数`      
- `直接净成交投入产出比 = 直接净成交金额 / 花费`  
- `深度互动率 = 互动量 / 观看次数 × 100%` 
- `新增粉丝成本 = 花费 / 粉丝关注量`
- `进店率 = 进店量 / 观看次数 × 100%`      
- `观看成本 = 花费 / 观看次数` 
- `直接订单成本= 花费 / 直接成交笔数`  
- `预售订单成本= 花费 / 总预售成交笔数`  
- `直接预售订单成本 = 花费 / 直接预售成交笔数`  
- `观看转化率= 总成交笔数 / 观看次数` 
- `观看率 = 观看次数 / 展现量`    
- `千次展现成本 = (花费 / 展现量) × 1000`  
- `点击单价 = 花费 / 进店量` 
- `互动成本 = 花费 / 互动量`
- `加购成本 = 花费 / 总购物车数`  
- `订单成本 = 花费 / 总成交笔数`

### 淘宝直播报表计算公式

- `商品点击成交转化率 = 成交人数 / 商品点击人数`
- `直播成交转化率 = 成交人数 / 观看人数` 
- `加购转化率 = 加购人数 / 商品点击人数` 
- `转粉率 = 新增粉丝数 / 观看人数` 
- `人均观看时长=观看总时长 / 观看人数` 
- `次均观看时长=观看总时长 / 观看次数`
- `商品点击率=商品点击人数 / 商品曝光人数` 
- `客单价 = 成交金额（元） / 成交人数` 
- `笔单价 = 成交金额（元） / 成交笔数` 
- `件单价 = 成交金额（元） / 成交件数` 
- `人均产出 = 成交金额（元） / 观看人数`
- `退款率 = 退款金额（元） / 成交金额（元） × 100%` 
- `退单率 = 退款笔数 / 成交笔数 × 100%` 
- `确认收货率 = 确认收货金额（元） / 成交金额（元） × 100%`

### 财务报表计算公式

- `财务口径收入 = 业务口径收入 / 1.06`  
- `业务口径收入 = 品牌费 + 切片 + 保量佣金 + 预估结算机构佣金 + 预估结算线下佣金`  
- `业务口径收入去折损 = 预估结算机构佣金 + 预估结算线下佣金 + 保量佣金 -（严选红包 + 直播间红包）`
- `财务口径收入去折损 = 业务口径收入去折损 / 1.06`
- `毛利率 = 毛利 / 财务口径收入`

### 跨报表计算公式

#### 字段口径说明

- `总成交笔数`：指超级直播报表中的「总成交笔数」
- `淘宝直播成交笔数`：指淘宝直播报表中的「成交笔数」
- `佣金净额`：指 `保量佣金 + 预估结算线下佣金 + 预估结算机构佣金 - 直播间红包 - 严选红包`

#### 日级计算公式

- `超级直播去退成交金额 = 总成交金额 × (1 - 退款金额（元） / 成交金额（元）)`
- `超级直播广告成交占比 = 总成交笔数 / 淘宝直播成交笔数`
- `广告收入 = 佣金净额 × (总成交笔数 / 淘宝直播成交笔数)`
- `盈亏平衡ROI = 广告收入 / 花费`

#### 周/月汇总规则

- `周广告收入 = Σ(可计算日广告收入)`
- `月广告收入 = Σ(可计算日广告收入)`
- `周盈亏平衡ROI = Σ(可计算日广告收入) / Σ(可计算日花费)`
- `月盈亏平衡ROI = Σ(可计算日广告收入) / Σ(可计算日花费)`

#### 汇总原则说明

- `广告收入` 为可汇总指标，先按日计算，再向周/月汇总
- `盈亏平衡ROI` 为比率指标，不可直接对日ROI求和或平均
- 周/月的 `盈亏平衡ROI` 必须使用“汇总后的广告收入 ÷ 汇总后的可计算日花费”计算
- 默认口径不使用“周/月聚合后的佣金净额 × 周/月聚合后的成交占比 ÷ 周/月花费”直接重算
- `业务口径收入去折损` 为财务侧净收入口径；`广告收入` 为投放分析口径下结合广告成交占比折算得到的归因收入指标，两者来源相关但用途不同，不应混用

#### 异常值处理规则

- 当 `淘宝直播成交笔数 <= 0` 或缺失时，当日 `广告收入` 与 `盈亏平衡ROI` 记为 `null` / `不可计算`，不得强制记为 `0`
- 周/月汇总时，仅汇总可计算日；不可计算日的 `广告收入` 与对应 `花费` 一并跳过
- 当 `花费 <= 0` 或缺失时，当日 `盈亏平衡ROI` 记为 `null` / `不可计算`
- 当 `佣金净额 < 0` 时，允许 `广告收入` 与 `盈亏平衡ROI` 为负值，不得截断为 `0`

### 跨报表数据关联规则

- `超级直播日期 = 淘宝直播日期 = 财务报表直播日期`
- `确保三个报表使用相同的时间周期进行分析`
- `淘宝直播粉丝数据 → 超级直播复购分析（关联逻辑待定）`

## 3、计划分类维度（plan_type）

基于计划名称中的关键词，自动归类为以下类型，用于维度分析。

### 分类规则


| plan_type | 关键词      | 说明       |
| --------- | -------- | -------- |
| 单品投放      | 含"单品"    | 单品推广类计划  |

1. 优先匹配"单品投放"（含"单品"即归此类）
2. 未匹配到"单品"的计划 → 默认归入"直播间投放"

### 派生维度定义


| 维度字段      | 来源表            | 说明                   |
| --------- | -------------- | -------------------- |
| plan_type | 从 plan_name 派生 | 计划分类标签（单品投放 / 直播间投放） |


### 代码实现说明

统一聚合引擎（period_resampler.py） 读取本规则后：

1. 扫描计划名字，含"单品" → 打标签 `单品投放`
2. 其余计划 → 打标签 `直播间投放`
3. 聚合时可用 `--dimension plan_type` 输出分类报表

## 四、数据文件自动扫描配置

### data_files 配置

`base_meta.yaml` 中新增 `data_files` 配置段，用于自动扫描目录下的数据文件并按关键词归类：

```yaml
data_files:
  super_live:
    keywords: ['超级直播']
  taobao_live:
    keywords: ['淘宝直播']
  financial:
    keywords: ['财务报表']
```

### 配置说明


| 字段            | 说明                             |
| ------------- | ------------------------------ |
| `keywords`    | 扫描目录时用于归类的关键词列表，支持多关键词         |
| `super_live`  | 超级直播数据文件（含"超级直播"关键词的 csv/xlsx） |
| `taobao_live` | 淘宝直播数据文件（含"淘宝直播"关键词的 csv/xlsx） |
| `financial`   | 财务报表数据文件（含"财务"关键词的 csv/xlsx）   |


### 工作原理

当 `--super-live` / `--taobao-live` / `--finance` 参数不传时，`generate_report.py` 自动扫描 `-d` 指定的数据目录，按关键词归类文件并按日期范围过滤。支持 csv 和 xlsx 两种格式，支持子目录递归扫描。

### 与旧方式的兼容

旧方式：手动传入文件名列表 `--super-live 2026-01超级直播.csv 2026-02超级直播.csv`
新方式：不传文件名参数，自动扫描目录按关键词归类

两种方式都支持，手动传入优先级高于自动扫描。
