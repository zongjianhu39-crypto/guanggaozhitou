# 交个朋友·广告智投工作台 — UI 重设计方案
**版本**: 1.0 | **日期**: 2026-04-30 | **状态**: 待审核

---

## 1. 设计理念

### 核心原则
1. **角色优先**：每个界面都要服务于特定用户的实际任务。如果设计对角色任务没有帮助，就是错误的。
2. **信号优于噪音**：最重要的信息2秒内就能看到。次要信息点一次就能找到。
3. **清晰不杂乱**：每个页面只回答一个问题。如果同时回答两个，就拆分开。
4. **一致性**：同一套设计系统、同一套设计变量、同一套组件库。无例外。

### 保持不变的部分
- 紫色渐变品牌识别（保留品牌认知度）
- 卡片式布局结构（用户已经熟悉）
- Supabase + 飞书认证栈（已经跑通）

### 需要改变的部分
- 信息架构 — 按角色任务流程重新组织
- 布局系统 — 侧边栏 → 角色专属工作区
- 数据密度 — 每屏显示更多数据，但层次更清晰
- 组件库 — 一致、可访问、可组合的设计变量

## 3. 布局架构

### 全局布局（所有页面）
```
┌─────────────────────────────────────────────────────┐
│  侧边栏 (240px 固定)    │   主内容区                 │
│  ─────────────────────  │  ─────────────────────────│
│  Logo + 品牌           │  页面标题                 │
│  ─────────────────────  │  (标题 + 面包屑)            │
│  角色选择器            │  ─────────────────────────│
│  ─────────────────────  │  内容区                   │
│  导航组                │  (角色专属)                │
│    · 监控              │                            │
│    · 分析              │                            │
│    · 归档              │                            │
│    · 配置              │                            │
│  ─────────────────────  │                            │
│  用户个人资料         │                            │
└─────────────────────────────────────────────────────┘
```

### 侧边栏设计（全新）
- 深色背景（#1E1B3A）作为视觉锚点和对比
- Logo区域带品牌渐变点缀
- **角色选择器**：投手/运营/管理者切换 — 高亮当前角色对应的导航项
- **导航组**：按工作流程步骤组织（监控 → 分析 → 归档 → 配置），而不是按页面
- **激活状态**：左侧紫色边框 + 背景色调
- **用户区域**：头像、名字、快速退出 — 始终可见于底部

### 页面标题设计
```
┌────────────────────────────────────────────────────┐
│  [← 返回?]        页面标题              [操作按钮]  │
│  副标题 / 上下文说明文字                         │
└────────────────────────────────────────────────────┘
```
- 页面标题：24px，半粗体，左对齐
- 上下文：13px，次要色，描述显示内容
- 操作按钮：右对齐，主要操作是填充按钮，次要是图标按钮或文字链接

### 响应式策略
- **桌面（1280px+）**：完整侧边栏 + 内容区
- **平板（768-1279px）**：可折叠侧边栏（仅图标模式），悬停展开
- **手机（< 768px）**：底部导航栏替换侧边栏

## 4. 组件库

### 4.1 导航组件

#### 侧边栏导航项
```
状态：默认 | 悬停 | 激活 | 禁用
- 默认：图标 + 文字，次要色
- 悬停：bg-sidebar-hover，白色文字
- 激活：左侧边框3px brand-primary，bg rgba(107,92,231,0.15)，白色文字
- 徽标：红点表示未读/新（可选）
```

#### 角色选择器
```
[🎯 投手] [📊 运营] [👁 管理者]
- 胶囊标签，水平布局
- 激活：品牌渐变背景，白色文字
- 未激活：透明背景，淡白色文字，悬停时显示边框
- 切换角色更新侧边栏高亮 AND 主内容区上下文
```

### 4.2 数据展示组件

#### 指标卡片（KPI展示）
```
┌─────────��───────────────────────┐
│  标签              [趋势箭头]    │
│  ─────────────────────────────── │
│  数值（大号，加粗）               │
│  ─────────────────────────────── │
│  对比 / 变化                     │
└─────────────────────────────────┘
变体：中性 | 正向（绿色） | 负向（红色） | 警告（琥珀色）
- 使用颜色语义，不只是文字
- 趋势：↑ 绿色，↓ 红色 → 中性（ convention：ROI下降=坏，消耗上升超出预算=坏）
```

#### 数据表格
```
- 固定表头带排序指示
- 斑马行（微妙，--bg-base）
- 行悬停：高亮 + --border-subtle 边框
- 行内操作：悬停时出现（查看、编辑、删除）
- 空状态：插图 + 操作CTA
- 加载状态：骨架行（脉冲动画）
- 分页：右下角，显示 "1-20 / 149"
```

#### 图表卡片
```
┌─────────────────────────────────┐
│  标题           [时间范围选择]   │
│  ─────────────────────────────  │
│  [图表区域]                    │
│  ─────────────────────────────  │
│  图例（如需要）                │
└─────────────────────────────────┘
- 图表类型：折线（趋势）、柱状（对比）、甜甜圈（组成）
- 提示框：深色背景卡片显示数值
- 响应式：小屏可滚动，带触控友好的控制手柄
```

### 4.3 表单组件

#### 输入框
```
状态：默认 | 聚焦 | 错误 | 禁用 | 只读
- 默认：边框 --border-default，白色背景
- 聚焦：边框 --brand-primary，box-shadow --shadow-glow
- 错误：边框 --error，红色标签在下方，红色背景色调
- 标签：字段上方，13px，中等字重
- 辅助文字：字段下方，12px，次要色
- 必填：红色星号
```

#### 下拉选择
```
- 原生选择器样式 OR 自定义下拉菜单
- 长列表（> 10项）带搜索过滤
- 多选带标签徽章变体
```

#### 按钮
```
变体：主要 | 次要 | 幽灵 | 危险
尺寸：sm (32px) | md (40px) | lg (48px)

主要：
  bg: 品牌渐变
  color: 白色
  悬停: brightness(1.05) + translateY(-1px) + 阴影
  点击: translateY(0) + 减少阴影
  禁用: opacity 0.5

次要：
  bg: 白色
  border: --border-strong
  color: --text-primary
  悬停: bg --bg-base

幽灵：
  bg: 透明
  color: --text-secondary
  悬停: bg --bg-base

危险：
  bg: --error
  color: 白色
  悬停: --error + brightness调整
```

#### 日期范围选择器
```
预设选项：今日 | 昨日 | 近7天 | 近30天 | 本月 | 上月 | 自定义
日历：范围选择的双月视图
时间：精确范围的可选时间选择器
```

### 4.4 反馈组件

#### 提示 / 通知
```
位置：右上角，堆叠
类型：success (green) | error (red) | warning (amber) | info (blue)
自动消失：5s（可调整），悬停暂停
操作：可选的内联操作链接
```

#### 加载状态
```
- 骨架：脉冲动画，匹配内容形状
- 加载中：仅用于小操作（< 5s预期）
- 进度条：已知时长的操作
- 全页加载：大加载中 + 上下文消息
```

#### 空状态
```
┌──────────────────────────────────────┐
│  [插图]                              │
│  标题：暂无数据                      │
│  说明：描述为什么没有 + 期望行为       │
│  [主要CTA] [次要CTA]                 │
└──────────────────────────────────────┘
```

### 6.1 引导流程（首次访问）
1. 角色选择屏幕（选择一个，后续可更改）
2. 所选角色工作区的快速浏览（3个屏幕，可跳过）
3. 降落在角色的首页

### 6.2 警告 → 分析 → 归档流程
```
1. ���告���现在订阅（高亮，脉冲动画）
2. 用户点击 → 内联展开显示上下文
3. "运行AI分析"按钮预填问题
4. 分析结果显示在GenBI面板
5. "保存到洞察"保存到归档
6. 标记警告为已解决
```

### 6.3 日期范围行为
- 默认："近7天"用于大多数视图
- "今日"用于实时仪表盘
- 日期选择器记住每个用户的最后选择
- URL参数编码日期范围以供分享链接

### 6.4 错误处理
- API错误：内联红色横幅带重试按钮（不清除表单）
- 认证错误：重定向到登录并保留返回URL
- 空状态：友好的插图 + 操作按钮

## 7. 无障碍（A11y）要求

- 所有交互元素键盘可访问（Tab、Enter、Escape）
- 焦点状态：2px实心brand-primary + 2px偏移
- 颜色对比：WCAG AA最低（文本4.5:1，UI 3:1）
- 所有纯图标的ARIALabel按钮
- 屏幕阅读器宣布：角色变化、警告到达、加载状态
- 动画：尊重 `prefers-reduced-motion`
